1.1 The Science of Rational Decision Making

Everyone has to take decisions. Sometimes decision making is a confusing process. Organizations usually face decisions that are much more complicated than individual decisions. Also, organizational decisions have a bigger impact and may affect many people.

Incorrect or non-optimum decisions can result in big penalties. For example, during World War II, mathematicians and other experts were tasked to find out ways to allocate resources to the war effort in the best way. Wrong decisions could mean being defeated in the war.

The discipline of Operations Research evolved over many decades in the 20th century, as experts tried to create concepts and tools to solve complicated decision processes in a rational and scientific way, helping to avoid waste, loss and failure to achieve goals. These techniques have found applications in all kinds of areas like management, healthcare, agriculture, manufacturing, defense etc.

1.2 What kind of decisions are we talking about?

In this context, we are only talking about situations that we can describe mathematically. So for example, this doesn't include ethical or moral decisions. It does include cases where the decision involves numbers that we have to choose, and where the consequences of our choice can be expressed as numbers. For example, if a factory produces A numbers of some product per week then one consequnce is that they may have to spend A x 1000 rupees per week to do so. So the effect of our number choices on the consequences should be such that we can express it mathematically in terms of functions. This is part of what we call our model of our problem.

By the way, the violet curve is an example of a non linear equation while the green line is an example of a linear equation.

Another example is that if we decide to eat 5 apples per week, we will get about 5 x 52 calories per week. Here apples per week is our decision, and the calories is the result of the decision. The relation between the decision and the consequence is our model of the situation. In this case it is a linear model which is described by a linear equation.

We need a certain number of calories in order to be healthy, which is our goal in this simple example. But too many calories can be bad for us, and we may end up spending more than we can afford if we eat too many apples. These are examples of constraints that we have to apply while choosing our decision values.

1.3 Linear Programming

We have noted that the apple consumption example involves linear relationships, i.e. it was a linear model. In the real world, we often see that different quantities are inter-related through quite complex mathematical functions which are not linear in general. Mathematical techniques do exist for these cases, but they tend to be difficult to understand and apply. Linear models are much easier in comparison to non-linear ones.

Linear Programming refers to the mathematical methods by which optimal values of certain quantities can be chosen, in order to get the best possible outcome, while ensuring that we meet certain specified goals, all the time assuming that the equations that describe the problem are all linear equations.

Although many real world problems involve non-linear functions, many practical problems can be represented approximately by linear equations, with some loss of accuracy. So linear programming has been found to be quite effective in practical situatiions.

Also, many concepts that we learn in linear programming will turn out to be relevant in non-linear programming as well, so linear programming is a stepping stone to begin understanding the more general types of operational research.

This is a set of interactive tutorials about Linear Programming. Most of these web pages will have a format similar to this one.
