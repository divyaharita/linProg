2.1 The Simplest Linear Equation!

We're going to assume that you have already studied linear equations, but we will quickily review what they are and look at some of their properties.

The simplest and the most familiar form of a linear equation is:

Y = A X

For example, we can say Y is the number of calories we get per week by eating apples. This is the product of calories per apple (A) and the number of apples we eat per week (X).

Here A is the slope of the equation. If we eat no apples, we get no calories -- so the graph passes through the origin. We say that the intercept is zero in this case.

Use the slider to change the number of calories per apple. As you would expect, the slope of the line changes accordingly. Take a minute to verify one or two examples against the graph -- for example try setting 60 calories per apple. Read off from the graph how many calories you would get if you ate 2 apples per week or 5 apples per week. Does the graph agree with your own expectation?

2.2 Line with Positive Intercept

Now let us assume that we are getting some calories from some other sources as well. Say we are going to get 200 calories from other sources besides apples. So even if we don't eat any apples, we still get 200 calories anyway -- and then we get more extra calories depending on the number of apples.

With this extra assumption, we can see that the line has an intercept of 200 calories. Its equation can be writtan as:

Y = A X + S

whrere A and S represent calories from apples and from other sources respectively.

2.3 Another form of the Linear Equation

We have just seen a simple and common form of writing a linear equation. But in our study of Linear Programming, we will usually write the linear equation in a different way. Let's introduce that other form by considering an example where we look at the calories that we get by eating apples and bananas.

Assume that we get 50 calories per apple and 90 calories per banana. We call these numbers A and B respectively.

Also, assume that we decide to eat X apples and Y bananas. How many calories would we get? Obviously, the total calories will be

Calories = A * X + B * Y

Assume that we want to eat apples and bananas in such a quantity that we get exactly 500 calories in total. In that case our equation becomes:

500 = A * X    +    B * Y

which becomes

50 * X    +    90 * Y   =  500

... which is shown in the chart as default. Try answering these questions by looking at the graph, assuming we need exactly 500 calories:

  • If we don't eat any apples, how many bananas do we need to eat?
  • If we don't eat any bananas, how many apples are needed?
  • If we eat 3 apples, how many bananas do we need?
  • If we eat 2 bananas, how many apples do we need?

2.4 Modifying the Calories Equation

Up to this point we assumed that apples contain 50 calories and bananas contain 90 calories. What if we change these assumptions, while keeping the requrement fixed at 500 calories? Let's try that now. Move the "Calories per Apple" slider from 50 to 70.

Observe how the graph changes when you go from 50 to 70 calories per apple and back again. How would you explain this change? Think for a moment and then click here:

As the number of calories per apple increases, the number of apples required for getting total of 500 calories, decreases. This applies also to the case where we eat no bananas at all (i.e. where all our calories come from apples). But that case is nothing but the X-intercept. Hence the X-intercept decreases if we get more calories per apple. The intercept is, in fact, inversely proportional to the calories per apple.

Similarly, try changing the "Calories per Banana" from 90 to 120. How would you describe the changes that you see? How would you explain the reason for this?
